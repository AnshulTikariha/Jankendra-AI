1. Introduction

1.1 Purpose

This document specifies the software requirements for Jankendra-AI, an AI-powered constituency intelligence platform designed to help elected representatives and their staff make informed, data-driven governance decisions.

1.2 Background

India's elected representatives — MLAs, MPs, and municipal councillors — govern millions of citizens using fragmented tools: WhatsApp groups, personal assistants with diaries, and spreadsheets that are perpetually outdated. Jankendra-AI addresses this infrastructure gap by providing institutional memory, intelligent issue clustering, development prioritization, and commitment accountability.

Problem Statement (India Innovates 2026 — Digital Democracy):
"Developing a secure hardware-software intelligence assistant that summarizes documents and meetings, drafts speeches and official responses, tracks constituency or community data, manages schedules, and provides real-time insights to support faster and more informed decision-making."

1.3 Definitions

WardSmallest administrative subdivision within a constituency
ClusterGroup of semantically similar complaints within the same ward

2. Scope

2.1 In Scope

  • Constituency data management (wards, demographics, infrastructure, schemes)
  • Citizen complaint intake and semantic clustering
  • Meeting transcript ingestion and commitment extraction
  • Weighted, prioritized to-do list for leaders and staff
  • Development project prioritization with multi-factor scoring
  • RAG-powered natural language assistant
  • Weekly governance digest (data-only, no editorializing)
  • Privacy-first local deployment architecture

2.2 Out of Scope (v1.0)

  • Physical hardware recording device (proposed for future)
  • Direct citizen-facing mobile app
  • Real-time WhatsApp integration
  • Electoral campaign / vote prediction features
  • Multi-constituency SaaS platform

9. Security & Privacy

Audio recordings → never leave the device Meeting transcripts → never leave the laptop Constituency data → never leave the laptop Complaint details → never leave the laptop Commitments → never leave the laptop AI responses → generated locally in production
  • Privacy enforced by architecture, not policy — no network connection during local operation
  • Static constituency data (DB1) never written to by AI — staff-only updates
  • Complaint audit trail (DB4) is immutable — no deletion
  • External Gemini API is development-only; production uses local LLM
  • Proposed hardware recorder: records only, no WiFi transmission during use
