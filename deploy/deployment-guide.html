1. Overview

Jankendra-AI runs as a monorepo with three components on a single GCP Compute Engine VM, fronted by nginx on port 80:

ComponentStackInternal portRole
FrontendReact + Vite (static build)—Citizen & staff UI, Google Maps
BackendFastAPI + SQLite8000Auth, wards, complaints, dashboard
AI ModulesFastAPI (separate service)8001Vertex AI text analysis
Design decision: AI modules run on a separate FastAPI process (port 8001), not inside the main backend. nginx routes /api/v1/analyze-text to the AI service.

2. Architecture

Internet :80
    │
    ▼
  nginx (/etc/nginx/sites-available/jankendra)
    ├── /                        → /opt/jankendra/frontend/dist  (React SPA)
    ├── /api/v1/analyze-text     → AI service :8001
    ├── /ai/*                    → AI service :8001 (docs, health)
    ├── /api/*                   → Backend :8000
    ├── /docs, /redoc, /health   → Backend :8000
    └── try_files → index.html   (client-side routing)

systemd services:
  jankendra-backend.service  → uvicorn app.main:app --port 8000
  jankendra-ai.service       → uvicorn ai_modules.api.main:app --port 8001

Data:
  SQLite DB  → /opt/jankendra/backend/data/jankendra.db
  Env files  → /etc/jankendra/{backend,ai,frontend}.env
  GCP creds  → /etc/jankendra/google-credential.json

3. What We Built

AI Text Analysis

  • ai_modules/analysis/text_analyzer.py — classifies complaint text
  • ai_modules/core/providers/vertex_gemini.py — Vertex AI Gemini provider
  • ai_modules/api/main.py — standalone FastAPI app
  • ai_modules/api/routes/analysis.py — POST /api/v1/analyze-text (no auth)

Model: gemini-2.5-flash in asia-south1 (Mumbai). gemini-2.5-pro is not available in that region.

Analysis returns: sentiment, severity, location, summary, keywords.

Deployment Infrastructure

  • deploy/scripts/install-vm.sh — one-time VM bootstrap (packages, nginx, systemd)
  • deploy/scripts/deploy.sh — repeatable deploy (pip, migrations, seed, frontend build)
  • deploy/scripts/deploy-from-windows.ps1 — one-click deploy from Windows
  • deploy/nginx/jankendra.conf — reverse proxy rules
  • deploy/systemd/*.service — backend & AI systemd units
  • deploy/env/*.env.example — production env templates
  • deploy/gcp/cloudbuild.yaml — GCP Cloud Build pipeline (primary CI/CD)
  • deploy/gcp/setup-cloud-build.sh — one-time Cloud Build + Secret Manager setup
  • deploy/gcp/submit-build.ps1 — manual Cloud Build from Windows

6. Deployment Steps (What We Did)

6.1 Local prerequisites (Windows)

  1. Installed Google Cloud SDK (gcloud) via winget
  2. Placed service account key at ai_modules/credentials/google-credential.json
  3. Ran gcloud auth activate-service-account --key-file=...
  4. Ran gcloud compute config-ssh for SSH/SCP access

6.2 Create VM & firewall

gcloud compute instances create jankendra-app \
  --project=project-d9125421-6cdc-4628-9f5 \
  --zone=asia-south1-a \
  --machine-type=e2-standard-2 \
  --tags=jankendra-app \
  --scopes=cloud-platform \
  --image-family=ubuntu-2204-lts \
  --image-project=ubuntu-os-cloud \
  --boot-disk-size=30GB \
  --metadata=enable-oslogin=FALSE

gcloud compute firewall-rules create jankendra-allow-http \
  --allow=tcp:80,tcp:443 \
  --target-tags=jankendra-app

6.3 Deploy from Windows (recommended script)

From repo root in PowerShell:

.\deploy\scripts\deploy-from-windows.ps1

This script:

  1. Authenticates with the service account key
  2. Creates VM + firewall if missing
  3. Packages repo into jankendra-deploy.tgz (excludes .git, node_modules, venvs, credentials)
  4. Uploads archive to /tmp/jankendra-deploy.tgz
  5. Uploads credentials to ~/google-credential.json (home dir — avoids /tmp permission issues)
  6. Strips Windows CRLF from shell scripts on the VM
  7. Runs install-vm.sh which calls deploy.sh
  8. Replaces YOUR_VM_EXTERNAL_IP in env files with actual IP

6.4 Manual steps we performed after first deploy

  • Installed Python 3.11 via deadsnakes PPA (Ubuntu 22.04 ships 3.10; project requires >=3.11)
  • Set VITE_GOOGLE_MAPS_API_KEY in /etc/jankendra/frontend.env and rebuilt frontend
  • Ran seed_demo_data.py to populate SQLite with demo users, wards, complaints, commitments

12. Troubleshooting

Frontend shows blank / Maps error

Set VITE_GOOGLE_MAPS_API_KEY in frontend.env and rebuild. Hard-refresh browser (Ctrl+Shift+R).

AI returns 502

Check Vertex AI credentials, project ID, and that gemini-2.5-flash is available in asia-south1.

Empty dashboard

Run seed_demo_data.py. Log in with demo phone + OTP 246810.

SSH / SCP fails

Ensure OS Login is disabled. Run gcloud compute config-ssh. Upload creds to home dir, not /tmp.

install-vm.sh: pipefail invalid

Windows CRLF line endings. Run: sed -i 's/\r$//' deploy/scripts/*.sh on VM before install.

Cloud Build console empty

Check project project-d9125421-6cdc-4628-9f5. Open History (not Triggers). Triggers is empty until GitHub is connected.

Wards not loading after city select

Select Bhopal for demo wards (W42–W47). Bengaluru/Kanpur need municipal sync. Run sync_municipal_wards.py on VM if needed.

Python version error

Ubuntu 22.04 has Python 3.10. Install 3.11 via deadsnakes PPA (handled in install-vm.sh).

If the script fails on the remote install step, use Option B2 below (manual commands).

Option B2 — Manual redeploy from Windows (direct SCP)

# PowerShell — run from repo root
$gcloud = "$env:LOCALAPPDATA\Google\Cloud SDK\google-cloud-sdk\bin\gcloud.cmd"
$proj   = "project-d9125421-6cdc-4628-9f5"
$zone   = "asia-south1-a"
$repo   = "C:\Users\bharath.g\Desktop\Janakendra\Jankendra-AI"
$archive = "$env:TEMP\jankendra-deploy.tgz"

# 1. Package repo
Push-Location $repo
tar -czf $archive --exclude=.git --exclude=node_modules --exclude=frontend/node_modules `
  --exclude=backend/.venv --exclude=.venv --exclude=backend/data/*.db `
  --exclude=ai_modules/credentials .
Pop-Location

# 2. Upload
& $gcloud compute scp $archive "jankendra-app:/tmp/jankendra-deploy.tgz" --zone=$zone --project=$proj

# 3. Extract + deploy on VM
& $gcloud compute ssh jankendra-app --zone=$zone --project=$proj --command=@"
sudo tar -xzf /tmp/jankendra-deploy.tgz -C /opt/jankendra
sudo find /opt/jankendra/deploy -type f \( -name '*.sh' -o -name '*.service' -o -name '*.conf' \) -exec sed -i 's/\r$//' {} +
sudo chown -R jankendra:jankendra /opt/jankendra
sudo bash /opt/jankendra/deploy/scripts/deploy.sh
"@

# 4. Rebuild frontend with Google Maps key (deploy.sh may miss env permissions)
& $gcloud compute ssh jankendra-app --zone=$zone --project=$proj --command="sudo -u jankendra env VITE_API_BASE_URL=/api/v1 VITE_GOOGLE_MAPS_API_KEY=YOUR_MAPS_KEY bash -c 'cd /opt/jankendra/frontend && npm run build'"

Write-Host "Done — hard-refresh http://34.180.31.35"

Replace YOUR_MAPS_KEY with your Google Maps API key.
